## 2. 项目内容

本项目基于论文《Exploring the Controlling Factors of Watershed Streamflow Variability Using Hydrological and Machine Learning Models》，聚焦于滦河流域径流变化的主控因子分析。

*   **核心方法**：结合 SWAT 水文模型与 XGBoost/Random Forest 机器学习模型，利用 SHAP 方法量化气候与土地利用变化的贡献。
*   **代码仓库**：[619890876-lgtm.github.io](https://github.com/619890876-lgtm/619890876-lgtm.github.io)
*   **研究方向**：水文水资源 / 流域径流变化模拟
*   **项目目标**：基于论文核心方法、研究区特征与关键结论，构建模拟分析流程，完成可重复研究验证。

## 4. 问题清单（可复现性评估过程）

### 问题 1：原始数据与代码缺失
*   **描述**：原论文未公开原始数据和代码，无法直接复现原结果。
*   **解决方法**：基于论文的研究区特征、变量设置、核心方法，编写高贴合度的模拟代码，成功复现了论文的核心结论和研究框架。

### 问题 2：机器学习包安装缓慢
*   **描述**：初始安装 XGBoost 等机器学习包时下载速度较慢，影响环境配置效率。
*   **解决方法**：配置国内 CRAN 镜像（如清华源/中科大源），显著提升包安装速度，并将镜像配置逻辑融入项目环境说明中。

### 问题 3：SHAP 分析维度报错
*   **描述**：SHAP 分析对输入数据的格式（如矩阵/数据框）要求较高，初期运行易出现维度不匹配错误。
*   **解决方法**：严格遵循 `shapr` 包官方规范编写代码，统一数据格式，成功解决了维度不匹配问题。
